## v1.2 — FIRST REAL USERS (Target: 2-3 weeks after deploy)

### Goal: 50 humans using it. Real feedback. Hype building.

### Feature 1: Invite Code System

**Schema addition:**
```sql
CREATE TABLE invite_codes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  code TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  max_uses INTEGER NOT NULL DEFAULT 50,
  current_uses INTEGER NOT NULL DEFAULT 0,
  created_by INTEGER REFERENCES users(id),
  source TEXT,
  active BOOLEAN DEFAULT 1,
  expires_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE user_invite_codes (
  user_id INTEGER REFERENCES users(id),
  code_id INTEGER REFERENCES invite_codes(id),
  used_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (user_id, code_id)
);
```

**Admin panel:**
- Create codes (name, quota, expiry)
- View usage per code
- Deactivate codes
- See which users came from which code

**Registration flow:**
- Landing page → "Enter invite code" field
- Valid code → proceed to register
- Invalid/exhausted code → "Join waitlist" option
- Waitlist = email/phone collection (simple table)

### Feature 3: Background AI Personalization (Haiku)

**How it works:**
1. User completes a run (Strava sync or manual log)
2. Server triggers Haiku evaluation:
   - Input: user profile + last 5 runs + current plan
   - Output: JSON with plan adjustments
3. Plan silently updates
4. Every Sunday: generate "Weekly AI Summary" card
   - "This week I adjusted your tempo pace down 5 sec/km because your heart rate was elevated. Your easy runs are on point. Next week: adding one more interval session — you're ready."

**What Haiku receives (system prompt):**
```
You are Sprint Society's training intelligence engine.

RUNNER: {name}, {age}, {gender}, VDOT {vdot}, Tier: {tier}
GOAL: {race_goal}
INJURIES: {injuries}
THIS WEEK'S RUNS: {runs_summary}
CURRENT PLAN: {plan_json}
TRAINING LOAD: acute={atl}, chronic={ctl}, ratio={acwr}
READINESS: {score}/100

Evaluate this runner's recent performance and output ONLY a JSON response:
{
  "plan_adjustments": [...],
  "pace_zone_changes": {...},
  "insight_text": "1-2 sentence summary for the user",
  "risk_flags": [...],
  "confidence": 0.0-1.0
}
```

**Prompt caching:** System prompt (minus this week's runs) is static → cache it. Saves ~50% tokens.

**Cost:** ~₹0.07 per evaluation. 50 users × 3 runs/week = ₹10.50/week total.

**User sees:**
- Training plan that feels alive (updates without them asking)
- Weekly summary card in their dashboard
- "AI-powered" badge on their plan

## Risk Register

| Risk | Impact | Mitigation |
|------|--------|------------|
| Railway free tier limits | App goes down | Upgrade to $5/mo immediately if needed |
