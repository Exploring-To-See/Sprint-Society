## 1. INFRASTRUCTURE — Deploy to Cloud

### 1.1 PostgreSQL Migration (Replace SQLite)

**Why now:** SQLite can't handle concurrent admin queries, remote access, or professional-grade data management.

**Action:**
- Provision PostgreSQL on Railway (free tier: 500MB, sufficient for 1000 users)
- Migrate schema from `better-sqlite3` → `pg` (node-postgres) or Prisma ORM
- All tables: users, runs, chat_messages, events, communities, challenges, etc.
- Enable automated daily backups (Railway provides this)
- Admin can connect via TablePlus/DBeaver from ANY machine

**Data access for admin:**
- Railway exposes PostgreSQL connection string
- Connect from any SQL GUI tool remotely
- Run queries: "show all runners inactive 7+ days", "export event attendance CSV"
- Full CRUD from admin panel (already built, just needs DB swap)

**Cost:** Free tier (500MB) → Paid ($7/mo) if exceeds

### 1.2 Railway Deployment

**Action:**

## 2. INVITE CODE SYSTEM — Phased Registration

### 2.1 How It Works

```
Public lands on app → Sees landing hero (4 slides) → "Join Sprint Society"
                                                           │
                                          ┌────────────────┼────────────────┐
                                          ▼                ▼                ▼
                                   Has invite code?   No code?        Waitlist
                                          │                │                │
                                          ▼                ▼                ▼
                                   Enter code →      "Join waitlist"   Gets notified
                                   Validate quota        (email/phone)  when slot opens
                                          │
                                          ▼
                                   Register → Onboarding → App
```

### 2.2 Invite Code Features

**Code structure:**
- Custom alphanumeric codes (e.g., RUNRAW, CRC, SPRINT50, ISHU-VIP)
- Each code has:
  - `name` — display name (e.g., "Run Raw Runners")
  - `code` — the actual code string
  - `max_uses` — quota (50, 100, 200)
  - `current_uses` — how many used
  - `created_by` — admin who created it
  - `expires_at` — optional expiry date
  - `tier` — what plan they get (free trial? starter?)
  - `source` — attribution (run club, influencer, organic)
  - `active` — can be deactivated instantly

**Admin panel management:**
- Create new codes (set name, quota, expiry)
- View dashboard: which codes are performing, conversion rates
- Deactivate codes instantly
- Extend quotas
- See which users came from which code
- Export attribution data

### 2.3 Phased Rollout Plan

| Phase | Size | Codes | Purpose |
|-------|------|-------|---------|
| Alpha | 50 | FOUNDER50 | Core team testing |
| Beta | 200 | RUNRAW, CRC, SPRINT100 | Run club partners |
| Early Access | 500 | Various influencer codes | Hype + feedback |
| Open | 1000+ | Public registration opens | Scale |

### 2.4 Waitlist

- Simple email/phone collection
- "You're #47 in line" — shows position
- Notify in batches when slots open
- Creates FOMO and demand signal

```
└─────────────────────────────────────────────────────────┘
```

### 4.2 Tier: ₹9/month — "Base" (Haiku, No Chat)

**What the user experiences:**
- Personalized training plan that updates silently after each run
- Weekly AI summary: "Here's what changed and why"
- Smart challenge selection based on their weaknesses
- Race predictions that actually reflect THEIR pace trends
- Notification: "Your AI coach adjusted your plan this week"

**What happens behind the scenes:**
- After each Strava sync/run log: Haiku evaluates new data
- Silently re-calculates pace zones, adjusts weekly plan
- Weekly: generates a visible summary card
- Haiku reads full user AI profile + recent run data
- Outputs structured JSON: updated plan, adjusted zones, insight text

**Token usage:** ~500 tokens input + 300 output per run evaluation  
**Monthly cost per user:** ~₹2-4/month  
**User perception:** "This app knows me. My plan is always fresh."

**What it does NOT include:**
- No chat interface
- No conversation with AI
- No diet/nutrition features
- No on-demand questions

### 4.3 Tier: ₹99/month — "Pro" (Sonnet, Structured Chat)

**Everything from ₹9 PLUS:**

**Scheduled Rituals (AI-initiated):**
- Pre-run check-in: "About to run? Quick check..." → MCQ options (How's energy? Any pain? Slept well?)
- Post-run check-in: "How was that run?" → Rate effort, note anything unusual
- Weekly review: AI summarizes week, asks 2-3 targeted questions to improve next week

**On-Demand Chat (User-initiated):**
- User opens chat anytime
- AI guides conversation with MCQ options + "Other" for free text
- Topics: training questions, race prep, injury concerns, motivation
- AI ALWAYS responds in context of their profile — never generic

**Interaction Format:**
```
┌─────────────────────────────────────────┐
│ Coach: How are your knees feeling after  │
│ yesterday's tempo run? (I noticed your   │
│ pace dropped in the last 2km)            │
│                                          │
│ ┌──────────┐ ┌────────────┐ ┌────────┐ │
│ │ All good │ │ Bit tight  │ │ Other  │ │
│ └──────────┘ └────────────┘ └────────┘ │
│                                          │
│ [Type your own response...]              │
└─────────────────────────────────────────┘
```

**AI Memory:**
- Every response stored and analyzed
- Key facts extracted and added to AI Profile
- "User mentioned left knee discomfort after tempo runs" → stored permanently
- Future training plans avoid consecutive hard knee-loading days

**Token usage:** ~1500 tokens input + 500 output per interaction (with context)  
**Monthly cost per user (daily active):** ~₹7-12/month  
**User perception:** "This coach REMEMBERS. It knows my knee acts up on Thursdays."

### 4.4 Tier: ₹199/month — "Elite" (Sonnet, Full Chat + Nutrition)

**Everything from ₹99 PLUS:**

**Full Two-Way Conversation:**
- Unlimited back-and-forth chat (not just MCQ)
- Can ask detailed, nuanced questions
- Discuss race strategy, mental preparation, cross-training
- AI can push back, challenge assumptions, give tough advice

**Nutrition AI Coach:**

**Core features:**
- Pre-run fuel recommendations (based on run type, duration, timing)
- Post-run recovery meals (protein window, glycogen replenishment)
- Race-day nutrition plan (carb-loading protocol, race morning, mid-race fuel)
- Periodized nutrition: macro ratios match training phase
  - Base phase: moderate carb, higher fat
  - Build phase: increasing carbs
  - Peak/taper: high carb, reduced fiber
  - Recovery: protein-focused

**Daily Tracking Dashboard:**

**How food logging works:**
- AI asks about meals in chat: "What did you have for lunch?"
- User describes in natural language: "Dal rice, curd, and a banana"
- AI estimates macros from description (knows Indian food)
- Shows estimated nutrition + asks for confirmation
- Dashboard updates in real-time
- No tedious manual entry — conversational logging

**Geographical food intelligence:**
- Indian food database: dal, roti, sabzi, rice, paratha, dosa, idli, upma, poha, etc.
- Understands regional cuisines (South Indian, Punjabi, Bengali, Gujarati, etc.)
- Knows restaurant food vs home-cooked portions
- Handles "I had 2 roti with paneer bhurji" → instant macro estimate

**Medical/personal considerations:**
- Iron recommendations for female runners
- Anti-inflammatory foods for injury-prone runners
- Electrolyte needs based on sweat rate + weather
- Vegetarian/vegan protein sources prioritized if applicable
- Conditions: diabetes, thyroid, PCOS → adjusted targets

**Runner-specific nutrition intelligence (what NO app offers):**
- "You have a long run tomorrow. Tonight eat complex carbs — dal-rice, sweet potato"
- "Post your tempo run, eat within 30 min: paneer + banana + milk"
- "Race in 3 days. Start carb-loading today: add 100g extra carbs to each meal"
- "Your iron was low this week. Add spinach/methi to dinner, pair with lemon for absorption"
- Hydration: "You ran 10K in 32°C — drink 500ml extra today + add electrolytes"

**Token usage:** ~2000 tokens input + 800 output per interaction  
**Monthly cost per user (daily active, 5 messages/day):** ~₹15-25/month  
**User perception:** "I deleted HealthifyMe. This one app does EVERYTHING for my running."

## 5. AI MEMORY SYSTEM — "My AI Profile"

### 5.1 What Gets Stored

**Permanent (never deleted unless user requests):**
- Injuries (past and current)
- Medical conditions
- Dietary restrictions/preferences
- Running goals (current race targets)
- Equipment (shoes, watch, what they own)
- Personal context (work schedule, family constraints)
- Food preferences (vegetarian, allergies, dislikes)
- Key conversation insights ("user gets anxious before races")

**Refreshed weekly:**
- Current VDOT, pace zones, fitness level
- Training load status (fresh/optimal/overreaching)
- Recent run performance trends
- Current training phase
- Nutrition adherence patterns

**Rolling context:**
- Last 10 chat messages (full text)
- Last 5 AI insights extracted from conversations
