# Machine Learning Systems (Reddi 2026)

A 1900-page Harvard textbook covering the *engineering* of ML systems — from data pipelines to deployment, hardware, MLOps, and responsible/sustainable AI.

## Identification
- **Author:** Vijay Janapa Reddi (Harvard SEAS)
- **Publisher:** Self-published (open), January 2026
- **Source:** [[../entities/book-mlsys]]

## Structure (21 chapters)

1. Introduction
2. **ML Systems** — deployment paradigms (Cloud, Edge, Mobile, Tiny ML, Hybrid), selection framework
3. DL Primer
4. **DNN Architectures** — MLPs, CNNs, RNNs (algorithmic structure + computational mapping + system implications)
5. **AI Workflow** — system thinking, decisions cascading, multi-timescale feedback
6. **Data Engineering** — four pillars, pipelines, ingestion (batch/stream, ETL/ELT), processing, [[../concepts/data-labeling]], storage, [[../components/feature-store]], data governance
7. **AI Frameworks** — TensorFlow / [[../entities/pytorch]] / [[../entities/jax]], computational graphs, [[../concepts/automatic-differentiation]], framework selection
8. **AI Training** — system architecture, distributed training
9. **Efficient AI** — efficiency principles
10. **Model Optimizations** — [[../components/quantization]], [[../components/pruning]], [[../components/knowledge-distillation]]
11. **AI Acceleration** — GPU/TPU/NPU hardware, accelerator design
12. **Benchmarking AI** — performance, fairness, robustness benchmarks
13. **ML Operations** — [[../concepts/mlops]] lifecycle
14. **On-Device Learning** — federated and on-device training
15. **Security & Privacy** — adversarial attacks, [[../components/differential-privacy]], [[../components/federated-learning]]
16. **Robust AI** — distribution shift, adversarial robustness
17. **Responsible AI** — fairness, transparency, accountability, [[../concepts/interpretability]]
18. **Sustainable AI** — energy, carbon, lifecycle
19. **AI for Good** — applications
20. **AGI Systems**
21. Conclusion

## Why It Matters
- The systems-engineering complement to algorithm-focused books like [[understanding-deep-learning]] and [[probabilistic-ml-introduction]]
- Use this for *productionization*: data pipelines, deployment, MLOps, hardware, governance
- Most recent (2026) treatment of LLMs, AGI systems, sustainability
- Covers everything the math-first books skip
